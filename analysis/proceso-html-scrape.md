1. **Pattern:** Greenshift uses per-element `<style>` blocks immediately before the element — a WordPress page-builder convention. Every block gets its own scoped CSS.
2. **Theming via CSS variables:** Colors use WordPress `--wp--preset--color--*` variables with hard fallbacks. This is the primary mechanism for theming.
3. **Background overlay pattern:** Uses a `div.gspb_backgroundOverlay` with `position:absolute; z-index:-1` + `isolation:isolate` on the parent to create stacked backgrounds.
4. **AOS integration:** Animation data attributes (`data-aos`, `data-aos-easing`, `data-aos-duration`) trigger Greenshift's animation system, which toggles CSS custom properties (`--gs-root-animation-*`) via JavaScript.
5. **Step cards are 2-col sub-rows** (col--2 + col--10) inside container wrappers — each badge is a separate column with z-index:1 for overlap effect.
